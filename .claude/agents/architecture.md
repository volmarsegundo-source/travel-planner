# 🏛️ Decisões de Arquitetura (ADRs)

## Como usar este arquivo
O **Arquiteto** documenta aqui todas as decisões técnicas relevantes no formato ADR.
Nunca delete um ADR — se uma decisão mudar, marque a anterior como "Substituído" e crie um novo.

---

## ADR-001: [Exemplo] Escolha do framework de backend
**Data**: 2025-01-01
**Status**: Aceito

### Contexto
Precisávamos escolher um framework para o servidor da aplicação.

### Decisão
Usaremos [framework X] pela familiaridade do time e suporte a TypeScript nativo.

### Consequências
**Positivas**: curva de aprendizado baixa, ecosistema maduro.
**Trade-offs**: menos performance bruta que alternativas mais próximas do metal.
