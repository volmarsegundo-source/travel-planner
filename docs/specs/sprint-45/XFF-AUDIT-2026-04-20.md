## Hipótese de manifestação em runtime

Em runtime Vercel (Serverless Function, Node runtime):
- `x-forwarded-for` tipicamente chega como `"<client_ip>"` (sem cauda de proxy).
- Em configurações com Cloudflare → Vercel ou quando `x-vercel-forwarded-for` é usado internamente, a cauda pode conter edge IPs que rotacionam por request.

Com o parser bugado, qualquer variação na string (cauda que aparece/some) cria uma chave Redis distinta. Efeito:
- **Rate limit por IP é bypassável** — o cliente consegue enviar N requests sem acumular no mesmo bucket.
- Comprovado em unit/smoke level: `tests/smoke/forgot-password-rate-limit.smoke.test.ts` cenário DIAGNÓSTICO (6 successes, 6 chaves Redis distintas).

**Confirmação em produção:** pendente (Probe-3 de SPEC-TEST-FORGOTPW-RATE-LIMIT-001 foi abortado por Redis down em Staging).

## Fix proposto (Wave 1)

### Criar helper centralizado

```ts
// src/lib/http/get-client-ip.ts
export function getClientIp(headers: Headers): string {
  return (
    headers.get("x-forwarded-for")?.split(",")[0]?.trim() ||
    headers.get("x-real-ip") ||
    "unknown"
  );
}
```

### Migrar os 4 sites

- Os 2 sites corretos (health, nextauth) passam a usar o helper por consistência.
- Os 2 sites bugados (auth.actions.ts:23, :90) passam a usar o helper e o bug some.

### Teste unitário

```ts
// src/lib/http/__tests__/get-client-ip.test.ts
describe("getClientIp", () => {
  it("returns leftmost IP from XFF chain", () => {
    const h = new Headers({ "x-forwarded-for": "200.150.100.50, 76.76.21.9" });
    expect(getClientIp(h)).toBe("200.150.100.50");
  });
  it("falls back to x-real-ip when XFF missing", () => {
    const h = new Headers({ "x-real-ip": "1.2.3.4" });
    expect(getClientIp(h)).toBe("1.2.3.4");
  });
  it("returns 'unknown' when neither header present", () => {
    expect(getClientIp(new Headers())).toBe("unknown");
  });
  it("trims whitespace in XFF chain", () => {
    const h = new Headers({ "x-forwarded-for": "  10.0.0.1  " });
    expect(getClientIp(h)).toBe("10.0.0.1");
  });
});
```
